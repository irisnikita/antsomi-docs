For development teams, regulations and rules are very important to ensure the effectiveness and efficiency of the team. Below are some regulations and penalties that could be applied in a Antsomi fullstack development team:

| Priority | Rule                                                                                                                                                                                | Penalty (Scar) |
|----------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------|
| 1        | Merge code makes error or lose someone else's code                                                                                                                                  | 1.5            |
| 1        | Automatically merge code into sandbox or production without mentor's approval                                                                                                       | 1.5            |
| 1        | Code on production environment makes critical bug: <br /> - Not validate variables (Cann't read property of undefined) <br /> - Make blank page error <br /> - UI Render many times | 1.5            |
| 2        | The function does not include a try-catch block                                                                                                                                     | 1              |
| 2        | Push console.log() to sandbox, production branches                                                                                                                                  | 1              |
| 3        | Late for work (On time at 8h30)                                                                                                                                                     | 0.5            |
| 3        | Not following coding conventions                                                                                                                                                    | 0.5            |
| 3        | Write new Component but do not write Component document                                                                                                                             | 0.5            |
| 3        | Late deadline without informing mentor                                                                                                                                              | 0.5            |

:::danger

If you have **five** scars, you will be responsible for inviting water for the entire team and will have the honor of being ranked on the hall of fame.

:::
